# 🐄 Cattle Health AI

A deep learning web application that predicts whether cattle are **Healthy** or **Possibly Sick** from images. Supports single image analysis with Grad-CAM visualization and batch folder analysis with summary reports.

## Installation

**1. Clone the repository**

```bash
git clone https://github.com/your-username/cattle-health-ai.git
cd cattle-health-ai
```

**2. Create a virtual environment (recommended)**

```bash
python -m venv venv

# On Windows
venv\Scripts\activate

# On Mac / Linux
source venv/bin/activate
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

The main packages used are:

| Package | Purpose |
|---|---|
| torch, torchvision | Model training and inference |

| seaborn | Confusion matrix plot |
| tqdm | Progress bars |
| jupyter | For running the EDA notebook (optional) |

## Dataset Setup

Create the `data/raw/` folder and organize your images into the following structure. Each view type (nose, body, mouth, foot) should be a subfolder under the class folder:

```
data/raw/
    healthy/
        nose/       ← nose closeup images of healthy cattle
        body/       ← full body images of healthy cattle
        mouth/      ← mouth images of healthy cattle
        foot/       ← foot images of healthy cattle
    possibly_sick/
        nose/
        body/
        mouth/
        foot/
```

**Important rules:**
- Every view folder must exist in **both** classes. If a view only exists in one class (e.g. you only have sick foot images), the preprocessing script will detect this, warn you, and automatically skip that view from training. This prevents the model from learning view type as a shortcut for the label.
- Supported image formats: `.jpg`, `.jpeg`, `.png`, `.webp`
- Aim for at least 50–100 images per view per class for reasonable results. More is always better.

## Preprocessing

Once your raw images are in place, run preprocessing. This does three things automatically:

1. **Cleans** — removes blurry and unreadable/corrupted images
2. **Balances** — caps all view types to the same count so no single view dominates training
3. **Splits** — divides each view independently into 80% train / 20% val

```bash
cd src
python preprocessing.py
```

**Example output you will see:**

```
── View validation ─────────────────────────────────
  healthy: ['body', 'foot', 'mouth', 'nose']
  possibly_sick: ['body', 'foot', 'mouth', 'nose']

  ✅ All views present in every class: ['body', 'foot', 'mouth', 'nose']

[healthy] View counts before balancing:
   nose: 120 images
   body: 340 images
   mouth: 98 images
   foot: 87 images
   → Capping all views to 87 images each

── Final dataset summary ───────────────────────────
  train/
    healthy/        →  278 images
    possibly_sick/  →  278 images
  val/
    healthy/        →  70 images
    possibly_sick/  →  70 images
```

You can adjust the blur threshold and val ratio at the bottom of `preprocessing.py` if needed:

```python
clean_dataset(src_dir="data/raw", dst_dir="data/processed", blur_threshold=100.0)
train_val_split(processed_dir="data/processed", train_dir="data/train", val_dir="data/val", val_ratio=0.2)
```

## Training

```bash
cd src
python train.py
```

Training configuration is at the top of `train.py`:

```python
CONFIG = {
    "model_name":  "mobilenet_v2",   # see Switching Models section
    "data_dir":    "data",
    "model_dir":   "models",
    "epochs":      150,
    "batch_size":  32,
    "lr":          1e-4,
    "num_classes": 2,
}
```

During training you will see per-epoch loss and accuracy for both train and val. The best model (highest val accuracy) is automatically saved to `models/best_model.pth`. Training curves are saved to `models/training_curves.png` when training finishes.

## Evaluation

After training, run evaluation against the val set:

```bash
cd src
python evaluate.py
```

This prints a full classification report (accuracy, precision, recall, F1) and saves a confusion matrix to `models/confusion_matrix.png`.

## Web App Features

### Home Page

The landing page presents two modes. Choose based on whether you want to analyse one image or many at once.

---

### Single Image Analysis

**How to use:**
1. Click **Single Image** on the home page
2. Click the file picker and select any cattle image (nose, body, mouth, or foot)
3. Click **Analyse**

**What you get:**
- ✅ **Healthy** or ⚠️ **Possibly Sick** label shown clearly in green or red
- **Confidence percentage** — how certain the model is about its prediction
- **Probability bars** — a visual bar for each class showing the exact probability
- **Grad-CAM heatmap** — a heatmap overlaid on your image showing exactly which part of the image the model focused on when making its decision. This is useful to verify the model is looking at the right things (e.g. the nose mucosa, not the background)
- **Original image** displayed below for comparison

---

### Batch Folder Analysis

**How to use:**
1. Click **Folder (Batch)** on the home page
2. Click the file picker
3. Navigate to your folder of images, press **Ctrl+A** (Windows/Linux) or **Cmd+A** (Mac) to select all images, then click Open
4. Click **Analyse All**

**What you get on the results page:**

**Summary box** at the top showing total images processed, how many are Healthy, and how many are Possibly Sick.

**Prediction Summary Chart** — a pie chart and bar chart side by side showing the health distribution across your entire batch. This is embedded directly in the page and can also be downloaded.

**Download buttons:**
- ⬇ **Download CSV** — a spreadsheet with one row per image containing: filename, prediction, confidence %, healthy probability %, sick probability %
- ⬇ **Download Summary Chart** — the pie/bar chart as a PNG file

**Individual Predictions Grid** — every uploaded image shown as a card with:
- A thumbnail of the image
- A **green border** if the model predicted Healthy
- A **red border** if the model predicted Possibly Sick
- The label (✅ Healthy / ⚠️ Sick) and confidence % shown below each image
- Filename shown on hover if it's too long to display

You can scroll through all images and immediately see at a glance which cattle are flagged as sick.

## Batch CLI (No Browser)

If you want to run batch predictions from the command line without the web app:

```bash
cd src
python batch_predict_visual.py /path/to/your/image/folder
```

Or run it without an argument and it will ask you to type the path:

```bash
cd src
python batch_predict_visual.py
# Enter folder path: /path/to/folder
```

This saves three files directly into your image folder:
- `batch_results_<id>.csv` — full prediction results
- `predictions_summary_<id>.png` — pie and bar chart summary

## Exploratory Data Analysis

Before or after preprocessing, you can explore your dataset visually using the notebook:

```bash
pip install jupyter   # if not already installed
jupyter notebook notebooks/eda.ipynb
```

The notebook has 5 sections:

1. **Class Distribution** — bar chart of image counts across train/val and both classes. Use this to confirm your dataset is balanced after preprocessing.
2. **View Distribution** — bar chart showing counts per view type (nose/body/mouth/foot) per class. Confirms the balancing step worked correctly.
3. **Sample Images per View** — a grid showing 3 example images for every class × view combination. Use this to visually confirm the right images ended up in the right folders.
4. **Pixel Intensity Distribution** — brightness histogram comparing healthy vs sick images. If the histograms look very different, that's a signal the model might be using lighting as a shortcut rather than real health features.
5. **Image Size Distribution** — histogram of original widths and heights before resizing. Useful to spot if your dataset has very inconsistent resolutions.

Run each cell with **Shift+Enter**.

## Switching Models

The project supports three backbone architectures. To switch, change `model_name` in `train.py`:

| Value | Model | Notes |
|---|---|---|
| `mobilenet_v2` | MobileNetV2 | Default. Fast, lightweight, good for CPU inference |
| `efficientnet_b0` | EfficientNet-B0 | Slightly more accurate, slightly slower |
| `resnet18` | ResNet-18 | Classic, reliable baseline |

After changing the model, retrain from scratch — the saved `best_model.pth` is architecture-specific and will fail to load if the architecture does not match.

## Notes

- The `data/` folder is not included in the repository. You must bring your own dataset and run preprocessing before training.
- The `models/best_model.pth` file is large (~9MB). If you want to include it in the repo, consider using [Git LFS](https://git-lfs.github.com/).
- The `app/static/uploads/` folder fills up over time as users upload images. You can safely delete its contents periodically — it does not affect the model.
- If Grad-CAM fails silently (no heatmap shown), make sure `pytorch-grad-cam` is installed: `pip install grad-cam`
